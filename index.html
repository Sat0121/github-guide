GitHub Pages ガイド

初心者のための完全ガイド：ローカル開発からデプロイまで

1. 開発環境の準備

必要なツール：

  • Git（バージョン管理システム）
  • テキストエディタ（Visual Studio Code推奨）
  • GitHubアカウント

Gitのインストールと初期設定：

  1. Windowsの場合：Git for Windowsからダウンロード・インストール
  2. Macの場合：ターミナルで以下を実行
    xcode-select --install
  3. 初期設定：
    git config --global user.name "あなたのGitHubユーザー名" git config --global user.email "あなたのメールアドレス"

2. GitHubでの準備

  1. GitHubにログイン
  2. 新しいリポジトリを作成（「+」→「New repository」）
  3. リポジトリ名を入力（例：my-website）
  4. 「Public」を選択
  5. 「Initialize this repository with a README」のチェックを外す
  6. 「Create repository」をクリック
重要： リポジトリ名は後でURLの一部となります。シンプルで覚えやすい名前を選びましょう。

3. ローカルプロジェクトの作成

プロジェクトフォルダの作成と初期化：

# プロジェクトフォルダの作成と移動 mkdir my-website cd my-website # Gitの初期化 git init

4. 基本的なファイル構成の作成

my-website/ ├── index.html ├── css/ │ └── style.css ├── js/ │ └── main.js └── images/

index.htmlの基本構造：

<!DOCTYPE html> <html lang="ja"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>My Website</title> <link rel="stylesheet" href="css/style.css"> </head> <body> <header> <h1>ようこそ</h1> </header> <main> <section> <h2>コンテンツセクション</h2> <p>ここにコンテンツを追加します。</p> </section> </main> <footer> <p>© 2025 あなたの名前</p> </footer> <script src="js/main.js"></script> </body> </html>

5. GitHubへのプッシュ手順

# ファイルをステージングに追加 git add . # 最初のコミットを作成 git commit -m "最初のコミット" # リモートリポジトリの追加 git remote add origin https://github.com/ユーザー名/my-website.git # mainブランチに変更 git branch -M main # GitHubにプッシュ git push -u origin main
注意：
  • URLは自分のGitHubリポジトリのものに置き換えてください
  • 初回プッシュ時にGitHubの認証が必要になります

6. GitHub Pagesの設定

  1. GitHubのリポジトリページに移動
  2. 「Settings」タブをクリック
  3. 左メニューから「Pages」を選択
  4. 「Source」で「main」ブランチを選択
  5. 「Save」をクリック
サイトが公開されるまで数分かかる場合があります。 公開後は https://ユーザー名.github.io/リポジトリ名 でアクセスできます。

よくあるトラブルシューティング

  • 認証エラー： GitHubの認証情報を確認し、必要に応じて個人アクセストークンを生成してください。
  • プッシュエラー： リモートリポジトリのURLが正しいか確認してください。
  • ブランチエラー：
    git branch
    で現在のブランチを確認し、必要に応じて
    git branch -M main
    を実行してください。

7. ページの更新方法

ウェブサイトの内容を更新する際は、以下の手順で行います：

1. ローカルでの更新

# 変更したいファイルを編集 # 例：index.htmlを編集した場合 git add index.html # 変更内容をコミット git commit -m "ページの内容を更新" # GitHubにプッシュ git push origin main

2. 複数ファイルの更新

# すべての変更をステージングに追加 git add . # 変更をコミット git commit -m "デザインとコンテンツを更新" # GitHubにプッシュ git push origin main
注意：
  • プッシュ後、GitHub Pagesの反映まで数分かかることがあります
  • ブラウザのキャッシュをクリアすると、更新後のページが表示されやすくなります
